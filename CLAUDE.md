# mini-3d-rpg（Glitch Survivors）— プロジェクトルール

3D Vampire-Survivors-like ゲーム（`index.html` 単一ファイル / three.js CDN / ビルド不要）。
ローカル起動は `Play RPG.bat` ダブルクリック、または `npx --yes serve -l 5599 .` → http://localhost:5599

---

## バックアップ運用ルール（最重要・絶対遵守）

### 修正するたびにやること

```
1. backups/ に「日付-連番-before」フォルダを作る
   例: backups/2026-06-23-1-before/
   → 修正前のファイルをコピーして入れる

2. 修正する

3. backups/ に「日付-連番-after」フォルダを作る
   例: backups/2026-06-23-1-after/
   → 修正後のファイルをコピーして入れる

4. before+after で1セット。セット数を数える
   → 7セット（14フォルダ）たまったら、削除前に必ずユーザーに確認を取る
   → 許可が出たら最新3セット（6フォルダ）だけ残し、古い4セット（8フォルダ）を削除
```

### デプロイするたびにやること（デプロイがあるプロジェクトのみ）

```
1. deployed/ に今日の日付フォルダを作る（例: deployed/2026-06-23/）
2. デプロイしたファイルを丸ごとコピーして入れる
3. 7個たまったら、削除前に必ずユーザーに確認 → 許可後、最新3個だけ残し古い4個を削除
```

### 注意
- `backups/` `deployed/` は **Git管理外**（.gitignore に追加済み）
- 削除は**必ずユーザーの確認を取ってから**。勝手に消さない

### 連番の決め方
- 連番は「その日付の中で」1から始める。同じ日に2回目の修正をしたら `-2-before` / `-2-after`。
- 日付が変わったらまた1から。
