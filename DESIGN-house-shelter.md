## 1. 中核となる状態モデル

プレイヤーに 1 つの状態を追加する。

```js
// player オブジェクトに追加
inHouse:false,        // 家（シェルター）の中にいるか
```

- **真になる条件**: レベルアップでカード選択を開く瞬間（`openLevelUp()`）。
- **偽になる条件**: カードを選び切って閉じる瞬間（`choose()` の最終分岐 / `mpDoCoopRestart()` / `die()` / `resetGame()`）。
- 連続レベルアップ（チェイン）中は `inHouse` は真のまま（家から出ずに続けて選ぶ）。

`game.picking`（build 22 で追加済み）と 1:1 で連動するが、`inHouse` は「安全・不可視・標的外」というゲーム的意味を持つ別概念として独立させる（将来「家に入るが選ばない」等に拡張しやすい）。

## 2. ネットワーク同期（⚠️最重要・地雷回避）

### 2.1 Firebase ルールの罠を踏まない

`rooms/$roomId/players/$slot` は**フィールド whitelist**（`x,z,ry,anim,hp,mhp,lv,alive,down,rp,t` のみ許可）。
ここに `inHouse` 等の**新フィールドを足すと、書き込み全体が `PERMISSION_DENIED` になり接続が再び全壊する**
（= 先日「全く繋がらない」を引き起こした原因そのもの。`down`/`rp` 追加でルール未更新だった件）。

➡ **`mpMyPayload()` には絶対に新フィールドを足さない。**

### 2.2 採用案：書き込み自由な `state/` パスで同期（ルール変更ゼロ）

攻撃同期（`state/atk_{slot}`）と同じ手口を使う。`state` は `.write:true` で子の制約なし。

```
rooms/{roomId}/state/house_p1   = { in:true,  t: <Date.now()> }   // host が家に入った
rooms/{roomId}/state/house_p2   = { in:false, t: <Date.now()> }   // guest は外
```

- **送信**: 自分の `inHouse` が変化した時だけ `state/house_{mySlot}` に set（毎フレーム不要＝低帯域）。
  退出時に `onDisconnect(... house_{mySlot}).remove()` も登録（落ちたら自動で「外」に戻す）。
- **受信**: `onValue(state/house_{otherSlot})` を listen → `MP.remoteInHouse`（bool）に反映。
  鮮度 `t` が古い（>5s 等）なら false 扱い（フリーズした相手をいつまでも不可視にしない）。

> これにより **ルール変更も再認証も不要**。接続を一切壊さずに R5/R6 を実現できる。

## 3. R3：家の中は安全（標的から外れる）

### 3.1 敵ターゲティング（host/solo・現状 1123-1132 行）

現状ロジック（host）：自分と相手（guest）の**近い方**を追う。

```js
// 現状（要約）
let tx=player.x, tz=player.z;
if(coop && guestが近い){ tx=guest.x; tz=guest.z; }   // 近い方を標的
```

**変更後**：在宅中のプレイヤーを**候補から除外**してから最近傍を選ぶ。

```js
// 疑似コード（host）
const meIn   = player.inHouse;                 // 自分が在宅
const otherIn= MP.remoteInHouse;               // 相手が在宅（state/house から）
// 標的候補リスト
const cands=[];
if(!meIn)    cands.push({x:player.x, z:player.z});
if(coop && !otherIn) cands.push({x:guest.x, z:guest.z});

if(cands.length===0){
  // 全員在宅 → 敵は「直前の標的方向にゆっくり前進」or その場で待機（§3.3）
} else {
  // cands の中で各敵にとって最近傍を選んで追尾（既存ロジックを候補限定で実行）
}
```

→ R3（自分が在宅なら敵は自分を追わない）と R5（相手だけ在宅なら全敵が自分へ）が**同じ 1 箇所**で両立する。

### 3.2 接触ダメージの無敵化

ターゲット除外だけだと、入室時に既に密着していた敵に殴られうる。**在宅中はその人に接触ダメージを与えない**。

- host/solo: `hurtPlayer` 呼び出し前に `if(player.inHouse) ない` ガード（1132 行 / flame 等の自傷は無関係）。
- guest: 同様に guest 側の被弾箇所（1122 行）に `if(player.inHouse) skip`。

### 3.3 全員在宅時の敵挙動（solo のレベルアップ＝ここに該当）

- **solo**: 在宅中は従来どおり**ゲームを一時停止**（`game.state='levelup'`）。敵は止まる。安全は pause で担保。
- **MP で両者同時在宅**（両方が同じ瞬間にレベルアップ）: 世界は動くが標的不在。
  → 敵は「最後の標的座標へ低速前進し、到達したら待機（ホップのみ）」。実装は §3.1 の `cands.length===0` 分岐で
  `e.group` を動かさない（その場待機）で十分。プレイ体験上、両者同時在宅は数百 ms で解消するので待機で問題なし。

## 4. R6：MP で家の中は相手から見えない

- 受信側（§2.2 の `MP.remoteInHouse`）が true の間、相手アバターを隠す：
  ```js
  if(MP.remote){ const hide = MP.remoteInHouse;
    MP.remote.group.visible = MP.remote.present && !hide; }
  ```
  （`mpTick` の remote 補間ブロック 1293-1301 付近に 1 行追加）
- 相手の**攻撃ビジュアル（弾・オーブ・炎）も隠す**：在宅中は攻撃していないので、送信側で在宅中は `mpBroadcastAttacks()` を送らない（空にする）→ 受信側は鮮度切れで自動消滅（既存 1500ms grace）。
- 名札（プレイヤーN）も一緒に消える（group の子なので `visible=false` で同時に消える）。

> 「消える」の見せ方（任意・决定事項②）：パッと消す／フェードアウト／煙パフ（`burst()` を一発）など。推奨＝**煙パフ＋フェード**で「家に入った」感を出す。

## 5. R2 & R4：家に入る演出と内装テクスチャー

ここが最大の分岐。**決定事項③**として 2 案を提示（推奨＝案A）。

### 案A（推奨）：オーバーレイ内装（軽量・堅牢）
- カード選択画面を「**家の内装**」テーマのパネルにする。背景に内装画像（暖色の木の室内＋窓から外が光る等）。
- フィールドのアバターは隠れ（不可視・無敵・その場停止）、カメラは**フィールドのまま**（MP では世界が動いているのが見える＝相手が戦っているのが分かる）。
- 内装の「テクスチャー」＝**オーバーレイ背景画像 1 枚**＋カードを木札風にスタイリング（CSS）。
- 長所: 実装が軽い／MP の非停止と相性が良い／カメラ遷移バグが無い。
- 短所: 「本当に部屋の中に入った」没入感は案B より弱い。

### 案B：本物の 3D 内装（没入・重い）
- 画面外に小部屋（床・壁 4 枚・窓・ドアの Three.js ジオメトリ＋テクスチャ）を建て、在宅中はカメラをそこへ移動。選択後フィールドへ戻す。
- 長所: 没入感が高い。短所: カメラ遷移／ライティング／MP で「自分は室内・相手はフィールド」を 1 シーンで両立する複雑さ／バグ表面積が大きい。

### テクスチャー（画像）について（重要・運用ルール準拠）
- **私（Claude）は画像を生成しない方針**（既定ルール）。内装テクスチャ画像は**GPT 用の画像生成プロンプトを私が書いて石井さんに渡し**、生成された画像をリポジトリに置いて読み込む。
- 画像を使わない場合は**コードだけ**で内装表現（CSS グラデーション＝木目調の色／3D は単色＋簡易シェーディング）も可能。**决定事項④**：画像（GPT 生成）を使うか、コード内装のみにするか。
- 案A 採用かつ画像使用なら、私が用意する内装プロンプト例（日本語＋英語、16:9、文字なし）を別途提示する。

## 7. レベルアップ・フロー統合（関数別の変更点）

| 関数 | 現状 | 変更 |
|---|---|---|
| `openLevelUp()` | `game.picking=true`、MP は上部カード/非停止、solo は `levelup` で停止 | 先頭で `enterHouse()` を呼ぶ（`player.inHouse=true`＋同期送信＋演出）。UI は「家の内装」テーマに。MP も solo も**安全な家**に入る |
| `choose()` 最終分岐 | UI 閉じ・`game.picking=false`・solo は `state='playing'` | `exitHouse()` を呼ぶ（`player.inHouse=false`＋同期送信＋復帰演出）。チェイン中は家に留まる |
| `die()` / `resetGame()` / `mpDoCoopRestart()` | picker クリア済み | `player.inHouse=false`＋`state/house_{slot}` を false で送信（在宅のまま死亡/再起動時の取り残し防止） |

新規ヘルパ：
```js
function enterHouse(){ player.inHouse=true; mpSendHouse(true);  /* 演出: 煙パフ＋アバター隠し（MP送信側） */ }
function exitHouse(){  player.inHouse=false; mpSendHouse(false); /* 復帰演出 */ }
function mpSendHouse(v){ if(MP.enabled&&MP.joined){ try{ MP._D.set(ref(`rooms/${MP.roomId}/state/house_${MP.mySlot}`),{in:!!v,t:Date.now()});}catch(e){} } }
```

`mpEnterRoom()` に listener 追加（§2.2）＋ cleanup（`MP._unsubHouse`）＋ `onDisconnect(... house_mySlot).remove()`。

## 8. 在宅中の入力・移動・武器

- **移動**: 在宅中は移動入力を無視（その場で安全に滞在）。1035-1050 行の player movement を `if(!player.inHouse){…}` で囲う。
- **武器発射**: 在宅中は停止（攻撃ビジュアルも送らない＝§4）。1062 行 `if(!player.down)` を `if(!player.down && !player.inHouse)` に。
- **カード操作**: タップ / J・K・L は従来どおり（build 22）。家の中での選択操作はこのキー/タップ。
- **solo**: `game.state='levelup'` で全停止（移動/武器も自然停止）。MP は上記ガードで「自分だけ停止」。

## 9. エッジケース表

| ケース | 挙動 |
|---|---|
| MP で自分が在宅中に相手が死亡 | 敵は全員自分を狙えない（自分在宅）→ §3.3 の待機。相手復活/再標的は相手退室で自然復帰 |
| 在宅中に連続レベルアップ | 家に留まり次のカードへ（`inHouse` true 維持） |
| 在宅中に同期リスタートが飛ぶ | `mpDoCoopRestart()` で `inHouse=false`＋送信、家 UI を閉じてフレッシュ開始 |
| ダウン（瀕死）中にレベルアップ | ダウン中は XP 取得しても `openLevelUp` を抑止（現状 `game.state!=='dead'` ガードに `&& !player.down` を追加検討）。要確認 |
| 相手のフリーズで `house_{slot}` が在宅のまま固着 | 鮮度 `t`>5s で false 扱い→アバター再表示（幽霊化防止） |
| host 自身が在宅 | 敵は guest を狙う（cands から自分除外）。guest 不在 solo なら pause |
| 全員在宅（同時 Lv up） | 敵その場待機、数百 ms で解消 |
