## Build 69 — 軽量化（発熱対策）

### 変更1: FPS上限60（タッチ端末のみ）★本命
- **問題**: `animate()` に上限が無く、120Hzスマホは毎秒120回描画＝GPU/CPU負荷2倍＝発熱。
- **場所**: `animate()`（旧 L2165）。
- **方法**: タッチ端末(`LOWP`)のとき、フレームを蓄積し約60fpsに間引く。dtは実経過を渡すので移動速度は不変。デスクトップは従来通り無制限。
  ```js
  const MAXFPS = LOWP ? 60 : 0;
  const MINFRAME = MAXFPS ? 1/(MAXFPS+6) : 0;   // +6=60Hz実機がジッタで間引かれない余裕
  let _frameAcc = 0;
  function animate(){ requestAnimationFrame(animate);
    let dt = clock.getDelta();
    if(MAXFPS){ _frameAcc += dt; if(_frameAcc < MINFRAME) return; dt = _frameAcc; _frameAcc = 0; }
    update(Math.min(dt,0.05)); renderer.render(scene,camera);
  }
  ```
- **効果**: 120Hz端末で描画回数 約半分。発熱の本命。
- **リスク**: 低。60Hz端末は実質変化なし。90Hz端末は45fps化（dt補正済みで体感スムーズ）。

### 変更2: クリスタルの点光源 約20個を削除 ★本命
- **問題**: `makeCrystal` が1個ごとに `PointLight` を追加。シーンに常時 約20個の動的ライト。WebGLは全ピクセルで全ライトを計算するため、全 `MeshStandardMaterial` のシェーダが重くなる。
- **場所**: `makeCrystal()`（旧 L649-655）。
- **方法**: `PointLight` 生成を**削除**。代わりに自己発光 `emissiveIntensity` を 0.45→0.9 に上げ、クリスタル自体は今まで通り光って見えるようにする（周囲の地面を照らす効果だけ失われるが影響は軽微）。
- **効果**: 全マテリアルのライト計算が激減。発熱のもう一つの主犯。
- **リスク**: 低〜中。周囲のライティングがやや平坦に。emissive増で見た目は維持。

### 変更3: 木・岩・クリスタルの影キャストを切る
- **問題**: 木 約136本＋岩＋クリスタルが毎フレーム影パスで再描画される（castShadow=true）。
- **場所**: `makeTree`(trunk/cone)、`makeRock`、`makeCrystal`。
- **方法**: これらの `castShadow=false`。**地面の `receiveShadow` とキャラ（プレイヤー/敵）の `castShadow` は維持**＝キャラの足元の影は残る。家(3軒)の影も維持（ランドマーク）。
- **効果**: 影パスの描画オブジェクトが激減。中〜大。
- **リスク**: 低。トップダウン視点で木の影はほぼ見えず、ゲーム性に無関係。

## Build 70 — バトルのマッチング信頼性（要・実機QA）

### 根本原因
auto-matchmaking のハンドシェイクがタイミング依存で脆い。中心は **"二重ホスト"**:
両クライアントが「自分が別々の部屋のホスト(p1)」になり、永遠にペアにならない → 開き直すと噛み合って繋がる。
- 待ち部屋ポインタ `lobby/open_battle` は **25秒で古い扱い**。スマホ待機中に画面が暗くなると心拍(heartbeat)が止まりポインタが古くなる → 後から来た側が2人目のホストになる。
- 二重ホストからの**自動復帰が無い**（=開き直すしかない）。
- 前回プレイの残骸（`go`フラグ等）が掃除されない。

### 変更4a: 二重ホストの自動解消 ★本命
ロビーで「一人ぼっちのホスト(p1)」のとき、ポインタを読んで**別の生存ホストが居れば、ID小さい方を残し大きい方が降格してゲスト参加**する（決定論的タイブレーク＝フラップしない）。
- 新関数 `mpHostReconcile()`(async): ポインタ取得 → `ptr.id !== 自分 && fresh && ptr.id < 自分のroomId && そのp1が生存(<12s)` なら降格参加。そうでなければ自分の部屋でポインタを(再)主張。
- 新関数 `mpJoinSpecificRoom(roomId)`: 自分のp1ノード削除＋自分のgoフラグ掃除 → role=guest/p2 に切替 → `mpEnterRoom()`（ゲストは go=true を立てるのでホストが開始）。
- **ロビー中のみ**実行（playing中は降格しない＝進行中ゲームを壊さない。ロビー中は敵も状態も無いので安全）。`MP._reconciling` で再入防止。
- 収束: 小ID側は降格せずポインタを主張し続け、大ID側がそれを見て1〜2心拍(2〜4秒)で降格参加。

### 変更4b: 復帰時の再マッチ（visibilitychange）
スマホがフォアグラウンド復帰した瞬間に、心拍(throttle)を待たず即再チェック。
- `visibilitychange` で `!document.hidden && MP.enabled && MP.joined` のとき: 自分のpresence再書込 → ロビー中なら相手スロット再読込（`mpOnRemote`）＋ホストは `go` を再取得して true なら `startGame()`。
- suspend中に取りこぼした onValue を回収する。

### 変更4c: 心拍間隔を短縮（3s→2s）＋ ポインタ窓を 25s→40s
- 心拍2sで待機中のポインタ更新を密に＝古くなりにくく。
- 窓40sは猶予拡大。生存ホストのpointer tsは常に2s以内なので、ゴーストルーム誤参加は既存の p1生存チェックが防ぐ（=安全）。

### 変更4d: 退出時の掃除（housekeeping）
`mpLeave()` で、自分が持つポインタを外し（`runTransaction`で id一致時のみnull）、自分の `lobby/go/<room>` を削除。残骸を減らす。
